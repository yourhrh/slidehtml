# UI Specification: 발표 모드

## 화면 구조

발표 모드는 전체 화면(fullscreen)으로 슬라이드 하나를 표시합니다.

```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│         [슬라이드 webview]           │
│       (화면 꽉 채움, 중앙 정렬)      │
│                                     │
│                                     │
│  ┌──────────────────────────────┐   │
│  │  1 / 3   (마우스 호버 시만)  │   │ ← 슬라이드 번호 오버레이
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

## 상태

| 상태 | 설명 |
|------|------|
| `currentIndex` | 현재 표시 중인 슬라이드 인덱스 (0-based) |
| `showOverlay` | 슬라이드 번호 오버레이 표시 여부 (마우스 hover) |

## 키보드 조작

| 키 | 동작 |
|----|------|
| `→` (ArrowRight) | 다음 슬라이드 |
| `Space` | 다음 슬라이드 |
| `←` (ArrowLeft) | 이전 슬라이드 |
| `ESC` | 발표 모드 종료 → 편집 화면으로 |

## 슬라이드 표시

- 슬라이드가 화면 전체를 채우도록 표시
- 비율 유지: `scale = min(w/slideW, h/slideH)` (SlideView와 동일 로직)
- 배경: 검정 (`#000`)

## 슬라이드 번호 오버레이

- 마우스 이동 시 3초간 표시 후 자동 숨김
- 위치: 우하단
- 내용: `{currentIndex + 1} / {slides.length}`
- `data-testid="present-overlay"`

## data-testid

| 요소 | testid |
|------|--------|
| 발표 모드 루트 | `present-screen` |
| 슬라이드 번호 오버레이 | `present-overlay` |
