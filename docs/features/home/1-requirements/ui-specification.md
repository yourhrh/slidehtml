# UI Specification: home

## 화면 개요
- **화면명**: 홈 화면
- **위치**: `App.tsx`의 screen 상태값 `'home'`
- **목적**: 슬라이드 프로젝트 폴더를 열거나 최근 항목을 재개할 수 있는 진입 화면

---

## 레이아웃

### 전체 구조
```
┌─────────────────────────────┐
│       앱 타이틀              │
│      "SlideHTML"            │
│                             │
│    [ 폴더 열기 버튼 ]        │
│                             │
│  최근 항목                   │
│  ┌────────────────────────┐ │
│  │ folderName  N개 · N분전 │ │
│  │ folderName  N개 · N분전 │ │
│  └────────────────────────┘ │
└─────────────────────────────┘
```

### 섹션별 구성

#### 1. 타이틀
- **위치**: 상단 중앙
- **내용**: "SlideHTML" 텍스트

#### 2. 폴더 열기 버튼
- **위치**: 타이틀 하단
- **동작**: 클릭 → `window.api.openFolderDialog()` 호출

#### 3. 최근 항목 목록
- **위치**: 버튼 하단
- **컴포넌트**: 각 HistoryItem을 행으로 표시
- **내용**: folderName, slideCount("N개"), lastOpened(formatRelativeTime)
- **빈 상태**: 히스토리 없으면 섹션 미표시

#### 4. 해상도 선택 모달 (신규 프로젝트 시)
- **조건**: openFolderDialog 또는 히스토리 클릭 후 hasConfig가 false일 때
- **내용**: 슬라이드 해상도 선택 (1280x720, 1920x1080, 1024x768)
- **확인**: 선택 후 `window.api.setupFolder()` 호출

---

## 상태별 UI

| 상태 | 표시 내용 |
|------|----------|
| 히스토리 없음 | 타이틀 + 폴더 열기 버튼만 |
| 히스토리 있음 | 타이틀 + 폴더 열기 버튼 + 최근 항목 목록 |
| 신규 프로젝트 | 해상도 선택 모달 오버레이 |

---

## 사용자 상호작용

| 요소 | 액션 | 결과 |
|------|------|------|
| 폴더 열기 버튼 | 클릭 | 폴더 다이얼로그 → hasConfig 확인 → 기존이면 편집 화면, 신규면 모달 |
| 히스토리 항목 | 클릭 | hasConfig 확인 → 기존이면 편집 화면, 신규면 모달 |
| 히스토리 항목 | 우클릭 | 컨텍스트 메뉴 표시 ("히스토리에서 제거") |
| 컨텍스트 메뉴 "히스토리에서 제거" | 클릭 | `removeHistory()` → 목록에서 제거 |
| 해상도 모달 확인 | 클릭 | `setupFolder()` 호출 → 편집 화면으로 전환 |
| 해상도 모달 취소 | 클릭 | 모달 닫기, 홈 유지 |
