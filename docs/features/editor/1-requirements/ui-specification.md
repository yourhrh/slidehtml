# UI Specification: editor

## 화면 개요
- **화면명**: 편집 화면
- **위치**: `App.tsx`의 screen 상태값 `'editor'`
- **목적**: 슬라이드 목록을 탐색하고 LLM이 생성한 슬라이드를 확인하는 화면

---

## 레이아웃

### 전체 구조
```
┌──────────┬──────────────────────────────┐
│ 사이드바  │     메인 슬라이드 뷰          │
│ (썸네일  │     (SlideView)              │
│  목록)   │                              │
│          │                              │
├──────────┴──────────────────────────────┤
│  ◁  2 / 5  ▷  | 발표 모드 | 터미널      │
└─────────────────────────────────────────┘
```

### 섹션별 구성

#### 1. 썸네일 사이드바
- **위치**: 좌측 고정 (스크롤 가능)
- **컴포넌트**: `Thumbnail` 목록
- **내용**: 슬라이드 번호 + webview 미리보기
- **선택 상태**: 현재 슬라이드 하이라이트

#### 2. 메인 슬라이드 뷰
- **위치**: 중앙 (나머지 공간 전체)
- **컴포넌트**: `SlideView`
- **내용**: 현재 선택된 슬라이드를 컨테이너 크기에 맞게 scale 조정

#### 3. 하단 툴바
- **위치**: 하단 고정
- **내용**:
  - 이전 버튼 (◁)
  - 슬라이드 번호 표시 "N / 전체" (e.g. "2 / 5")
  - 다음 슬라이드 버튼 (▷)
  - "발표 모드" 버튼
  - "터미널" 버튼

---

## 상태별 UI

| 상태 | 표시 내용 |
|------|----------|
| 슬라이드 없음 | 사이드바 빈 상태 + 메인뷰에 "슬라이드가 없습니다" 안내 |
| 슬라이드 있음 | 사이드바에 썸네일 목록 + 메인뷰에 현재 슬라이드 |

---

## 사용자 상호작용

| 요소 | 액션 | 결과 |
|------|------|------|
| 썸네일 | 클릭 | 해당 슬라이드로 이동 |
| ◁ 버튼 | 클릭 | 이전 슬라이드로 이동 (0에서 더 이상 안 감) |
| ▷ / "다음 슬라이드" 버튼 | 클릭 | 다음 슬라이드로 이동 (마지막에서 더 이상 안 감) |
| "발표 모드" 버튼 | 클릭 | Present 화면으로 전환 |
| "터미널" 버튼 | 클릭 | `window.api.openTerminal(folderPath)` |
| slides/ 파일 추가/삭제 | (chokidar push) | 슬라이드 목록 자동 갱신 |
